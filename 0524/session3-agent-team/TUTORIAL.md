# SESSION 3 튜토리얼: 에이전트 팀으로 일 시켜보기

**목표**: Claude 1개로 4명짜리 전문가 팀 효과 내기
**소요 시간**: 15~30분
**준비물**: [claude.ai](https://claude.ai) 계정 (무료 가능)

---

## 핵심 개념: Conductor 패턴

```
나 (Conductor/지휘자)
    ↓ 지시
[Researcher] → [Writer] → [Critic] → [Publisher]
    각자 역할 수행, 결과를 다음 에이전트에게 전달
```

**포인트**: Claude 1개지만 역할을 명시하면 각 "에이전트"처럼 다르게 작동한다.
비용: 무료 플랜도 가능. 긴 체인은 Claude Pro ($20/월) 권장.

---

## STEP 1: 기성 예제로 체험 (5분)

### 콘텐츠 팀 실행

1. [claude.ai](https://claude.ai) → 새 채팅 열기
2. 아래 전체 복사 → 붙여넣기 → 전송

```
당신은 4명 전문가 에이전트 팀의 Conductor입니다.
각 에이전트가 순서대로 작업하고, 결과를 다음 에이전트에게 넘깁니다.

[Researcher] 주제를 리서치해 핵심 포인트 5개와 SEO 키워드 10개를 뽑습니다.
[Writer]     Researcher 결과로 블로그 초안(600자)을 작성합니다.
[Critic]     Writer 초안의 개선점 3개를 구체적으로 제시합니다.
[Publisher]  Critic 피드백 반영해 최종본을 완성하고 Markdown으로 출력합니다.

주제: "1인 사업자가 AI로 고객 응대를 자동화하는 3가지 방법"

각 에이전트 앞에 [에이전트명] 태그를 붙여 순서대로 진행해주세요.
```

**확인할 것**: `[Researcher]`, `[Writer]`, `[Critic]`, `[Publisher]` 태그가 붙어서 순서대로 출력되는지.

---

## STEP 2: 다른 예제 2개 더 해보기 (선택)

### CS 팀 — 고객 이메일 자동 처리

```
당신은 고객 CS 에이전트 팀 Conductor입니다.

[Triager]   이메일을 읽고 유형 분류 (문의/불만/계약/스팸) + 우선순위 판단
[Responder] 분류 결과로 답장 초안 작성 (한국어, 공손하게)
[Escalator] 불만/계약 건은 담당자 에스컬레이션 메모 작성

입력 이메일:
"안녕하세요. 지난주에 신청한 컨설팅 일정이 아직 확정이 안 됐는데,
언제쯤 연락 받을 수 있을까요? 이번 주 안에 진행이 필요합니다."

각 에이전트 결과를 순서대로 보여주세요.
```

### 회계 팀 — 영수증 자동 정리

```
당신은 회계 처리 에이전트 팀 Conductor입니다.

[OCR]        영수증 텍스트를 파싱해 항목/금액/날짜를 추출합니다.
[Classifier] 비용을 카테고리(식비/교통/통신/장비/기타)로 분류합니다.
[Reporter]   월별 지출 요약표를 Markdown 표로 만듭니다.

영수증 데이터:
- 스타벅스 5/20 18,000원
- KTX 5/21 42,600원
- AWS 5/22 $23.40 (한화 31,800원)
- 쿠팡 모니터암 5/23 45,000원

처리해주세요.
```

## STEP 3: 내 업무 팀 직접 설계 ← 핵심 실습 (10분)

아래 템플릿에서 `[  ]` 부분만 채워서 전송:

```
나는 [내 직업/사업]을 운영하고 있습니다.
매주 반복하는 작업 중 AI 팀에게 맡기고 싶은 것: [반복 작업 설명]

이 작업을 처리할 에이전트 팀을 설계해주세요:
1. 필요한 에이전트 역할 3~4개와 각 역할의 책임
2. 에이전트 간 정보 전달 방식
3. 위 1~3번 양식처럼 실제로 바로 쓸 수 있는 Conductor 프롬프트

그 다음 실제로 그 프롬프트로 작업을 한 번 실행해주세요.
```

### 작성 예시

| 내 직업 | 반복 작업 |
|---------|---------|
| 유튜버 | 댓글 모아서 FAQ + 쇼츠 대본 뽑기 |
| 부동산 중개사 | 매물 정보 → 홍보 문구 + SNS 포스트 |
| 강의 판매자 | 수강생 질문 → 공지 + FAQ 문서 업데이트 |
| 쇼핑몰 운영자 | 상품 스펙 → 상세페이지 문구 3종 |
| 프리랜서 디자이너 | 클라이언트 브리핑 → 제안서 초안 |

## 에이전트 팀 설계 원칙 (참고)

**좋은 팀 구조**:
- 역할이 명확하게 분리됨 (Researcher ≠ Writer)
- 앞 에이전트 결과가 다음 에이전트 입력이 됨
- 3~5명이 적당 (너무 많으면 산만)

**역할 이름 예시**:
```
정보 수집: Researcher, Scout, Analyst
생성/작성: Writer, Creator, Designer, Coder
검토/평가: Critic, Reviewer, QA, Editor
출력/배포: Publisher, Formatter, Reporter
```

**확장 아이디어**:
- 이 프롬프트를 n8n 워크플로우에 넣으면 → 이메일 수신 시 자동 실행
- Claude API + Python으로 감싸면 → 웹훅으로 외부 시스템 연동
- Conductor 프롬프트를 저장해두면 → 매번 재사용 가능한 나만의 팀
