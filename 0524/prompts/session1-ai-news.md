## 프롬프트 A: 이번 주 AI 뉴스 내 사업 적용

```
이번 주 AI 업계에서 다음 뉴스들이 있었습니다:

1. Claude Code 한도 2배 — Pro/Max 사용자 피크타임 제한 제거. SpaceX 컴퓨트 제휴로 개인이 쓸 수 있는 AI 양 폭증.
2. MCP 표준화 — Stripe, Mastercard, Zapier, Salesforce가 AI 에이전트-도구 통신 표준(MCP) 채택. 에이전트가 결제·CRM·자동화를 직접 호출 가능.
3. AI 비용 표준 등장 — Helply: 티켓당 50¢, B2B SaaS 100+ 고객, 65% 자동 해결 보장. Salesforce: 엔지니어 1인당 연 $15K–20K AI 비용 = 급여의 4%.
4. 코딩 도구의 진화 — Cursor/Claude Code 등이 단순 자동완성이 아닌 "AI 팀 디스패치"로. Background Agent, Spec-Driven IDE 등장.

[내 정보]
- 직업/사업: (예: 프리랜서 마케터, 온라인 강의 운영자, 회계 컨설턴트)
- 현재 반복하는 업무: (예: 콘텐츠 작성, 고객 응대, 보고서 작성)
- AI 도구 사용 수준: (예: ChatGPT 가끔 씀 / Claude 매일 씀 / 거의 안 씀)

위 뉴스 4가지를 내 직업/사업 맥락에서 해석해주세요.
각 뉴스가 내 일상 업무에 어떤 의미인지, 지금 당장 뭘 바꿀 수 있는지 알려주세요.
```

## 프롬프트 B: AI 예산 계산기

```
AI 비용 벤치마크: 엔지니어 급여의 4% = AI 예산 (Salesforce 기준)

[내 정보]
- 월 수입 또는 사업 매출: (예: 월 500만원)
- 현재 AI 구독 비용: (예: Claude Pro $20/월, 없음)
- 주요 작업: (예: 글쓰기, 디자인 의뢰, 고객 응대)

내 상황에서:
1. "4% 룰" 적용 시 적정 AI 예산은 얼마인지
2. 지금 구독 중인 AI 도구가 적정한지 (부족/적당/과한지)
3. 50¢/티켓 모델처럼 내 업무를 "결과 단위"로 쪼개면 어떤 자동화가 가능한지
알려주세요.
```

## 프롬프트 C: 내 업무의 MCP 연결 상상하기

```
MCP(Model Context Protocol)가 표준화되면 AI 에이전트가 외부 도구를 직접 호출할 수 있습니다.
현재 Stripe(결제), Salesforce(CRM), Zapier(자동화), Mastercard(금융)가 연동됩니다.

[내 정보]
- 내가 매일 쓰는 도구: (예: 네이버 스마트스토어, 카카오톡 채널, 노션, 구글시트)
- 가장 번거로운 반복 작업: (예: 주문 확인 후 배송 메시지 보내기, 문의 답변 복붙하기)

만약 AI 에이전트가 내 도구들을 직접 연결해서 자동으로 처리할 수 있다면,
어떤 워크플로우가 가능한지 3가지 시나리오를 그려주세요.
지금 당장 실현 가능한 것과 1년 후 가능한 것을 구분해서요.
```
